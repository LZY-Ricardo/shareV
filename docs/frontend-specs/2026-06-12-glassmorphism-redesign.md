# shareV 玻璃拟态视觉重设计 前端规格

**日期：** 2026-06-12
**状态：** 草稿
**原始需求：** 对项目所有页面进行美化，从当前赛博朋克霓虹终端风格切换为 Glassmorphism（玻璃拟态）风格，进行全面视觉升级和细节打磨。

## 1. 背景与目标

当前 shareV 采用 "Neon Terminal" 赛博朋克深色主题（网格背景、扫描线、霓虹发光），视觉效果偏硬核技术感。用户希望切换为更柔和现代的 Glassmorphism 风格，提升整体精致度和视觉层次。

目标：
- 所有页面统一迁移到玻璃拟态设计语言
- 提升卡片、按钮、输入框、模态框等组件的质感
- 切换为浅色主题，用白色毛玻璃 + 柔和渐变光斑 + 微投影替代霓虹发光
- 优化间距、圆角、排版等细节
- 保持所有现有功能不变，纯视觉层改动

## 2. 用户与场景

所有现有用户场景不变。页面入口不变。

## 3. 范围

### 包含
- `style.css` 主样式表全面重写（配色、变量、组件样式）
- `admin.css` 管理页样式适配
- 4 个 HTML 页面的少量结构微调（如有必要）
- `app.js` 中 QR 码生成颜色调整（前景色适配新风格）
- 所有页面的背景效果替换（网格 → 柔和渐变光斑）
- 扫描线效果移除
- 所有组件样式更新：卡片、按钮、输入框、模态框、Toast、进度条等
- 响应式断点优化
- 动画效果从"硬核脉冲"改为"柔滑过渡"

### 不包含
- 不改变任何 JS 业务逻辑
- 不新增功能或页面
- 不修改服务端代码
- 不引入新的外部 CSS 框架或组件库
- 不修改 QRious 库
- 不修改 HTML 中教程文案内容

## 4. 页面入口与导航

不变。

## 5. 用户流程

不变。

## 6. UI 与交互规格

### 6.1 设计语言

**从 → 到：**

| 属性 | 当前（赛博朋克） | 目标（玻璃拟态） |
|---|---|---|
| 背景 | 纯深色 + 网格线 | 浅灰白底 + 柔和彩色渐变光斑 |
| 装饰 | 扫描线动画 | 无扫描线，用背景光斑代替 |
| 卡片 | 半透明深色 + 霓虹边框 | 白色毛玻璃（backdrop-filter: blur）+ 微投影 |
| 发光 | 霓虹 glow（box-shadow 发散） | 柔和投影 + 微妙彩色阴影 |
| 按钮 | 霓虹渐变 + 发光 hover | 白/彩色毛玻璃按钮 + 柔和 hover 浮起 |
| 输入框 | 深色 + 霓虹聚焦边框 | 白底/微透明白 + 柔和聚焦边框 |
| 颜色 | 高饱和 cyan/amber/green | 浅色系柔和蓝紫/珊瑚/薄荷 |
| 圆角 | 8-12px | 14-20px（更圆润） |
| 动画 | 硬核脉冲 + 扫描 | 柔滑淡入 + 微浮起 |
| 字体 | JetBrains Mono 为主 | Outfit 为主标题，JetBrains Mono 为数据 |

### 6.2 色彩体系（浅色系）

```
/* 背景 */

--bg-card: rgba(255,255,255,0.55) /* 卡片：半透明白 */
--bg-card-hover: rgba(255,255,255,0.75)
--border: rgba(255,255,255,0.5)   /* 柔和白色边框 */
--border-bright: rgba(255,255,255,0.7)

/* 主色：柔和蓝紫 */
--primary: #6c5ce7               /* 主色-蓝紫 */
--primary-dim: rgba(108,92,231,0.10)
--primary-glow: rgba(108,92,231,0.25)

/* 辅色 */
--accent: #e17055                 /* 珊瑚暖橙 */

--success-dim: rgba(0,184,148,0.10)
--danger: #d63031                 /* 柔红 */

/* 文字 */

--text-bright: #1a1d21            /* 近黑，标题 */
--text-dim: #8c95a1               /* 灰，次要文字 */
```

### 6.3 背景效果

移除 `.grid-bg` 网格背景和 `.scanline` 扫描线。替换为：

- 页面底色为浅蓝灰（#f0f2f7 → #f5f7fb 渐变）
- 2-3 个大面积柔和彩色光斑（position: fixed, filter: blur(80-120px)），缓慢飘动动画
- 光斑颜色：蓝紫 + 珊瑚 + 薄荷，低透明度（8-15%）在浅底上形成微妙色彩层次

### 6.4 卡片组件

```css
.card, .chart-section, .config-section, .node-bar {
  background: rgba(255, 255, 255, 0.55);
  backdrop-filter: blur(20px);
  -webkit-backdrop-filter: blur(20px);
  border: 1px solid rgba(255, 255, 255, 0.5);
  border-radius: 18px;
  box-shadow: 0 4px 24px rgba(0, 0, 0, 0.05);
}
```

### 6.5 按钮系统

- 主按钮：毛玻璃底 + 主色渐变 + 柔和 hover 放大
- 次要按钮：半透明底 + 彩色文字 + hover 加亮
- 危险/操作按钮保持对应语义色

### 6.6 导航/Header

- Logo 区域：去掉方括号 `[ ]`，改为更圆润的展示
- 副标题用更淡的颜色

### 6.7 教程页（guide.html / guide-clash.html）

- 步骤编号（.step-no）：从渐变方块改为圆形毛玻璃 + 主色数字
- 平台卡片：更大的圆角 + 柔和 hover 浮起
- 提示框（.step-tip）：从琥珀色警告改为柔和的半透明提示

### 6.8 管理页（admin.html）

- 侧边栏用户列表：白色毛玻璃底 + 选中态主色柔和背景
- 排行榜条形图：用主色渐变替代霓虹色
- 详情面板：更大圆角 + 内间距调整

### 6.9 动画

- 移除脉冲动画（pulse）
- 保留淡入动画但缓动更柔和（ease-out → cubic-bezier(0.16, 1, 0.3, 1)）
- 背景光斑缓慢飘动（20-30s 周期）
- 卡片 hover：translateY(-3px) + 微妙阴影增强

### 6.10 响应式

保持现有断点逻辑，调整小屏幕下的间距和圆角适配。

## 7. 状态覆盖

所有交互状态保持不变，仅调整视觉表现：

- **默认态：** 毛玻璃底 + 微投影
- **hover 态：** 背景加亮 + 微浮起 + 投影加深
- **active/选中态：** 主色边框 + 主色微发光
- **加载态：** 保持 loader-bar，颜色改为主色
- **错误态：** 柔红色替代当前红色
- **Toast：** 毛玻璃底 + 对应语义色文字 + 柔和投影

## 8. 数据与接口需求

无新增接口。QR 码生成颜色需在 JS 中调整：

- `foreground` 从 `#000000` 改为 `#2d3436`（深灰，匹配新文字色）
- `background` 保持 `#ffffff`

## 9. 前端实现提示

- 主要改动集中在 `style.css`（约 80% 改动量）和 `admin.css`（约 60% 改动量）
- `app.js` 中 QR 码颜色调整（约 3 处）
- `guide.js` 和 `guide-clash.js` 中 QR 码颜色调整（各 1 处）

- 背景光斑需要在 body 或 `.grid-bg` 替换元素上实现
